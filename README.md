# CS---360-Final-Project

**App Summary and User Needs**
For this project I created a simple mobile app that lets users log in and store information using a small local database. The goal was to make something practical that shows I understand the full app process from design to coding. The main user need was having an easy way to sign in, save data, and come back later without losing anything. I focused on keeping it simple and clear so anyone could understand how to use it without confusion.

**Screens and UI Design**
The app needed a login screen, a main screen, and screens for viewing or adding data. I designed the UI first in Project Two so I had a clear plan before coding. I kept the layout simple, with clear buttons and readable text so users would not get lost. This helped make the app feel organized and user friendly. I think the design worked well because everything is easy to find and does exactly what the user expects.

**Coding Approach**
When coding the app, I worked step by step. I focused on one feature at a time like login first, then database storage, then navigation between screens. I used what we learned in class about organizing code into sections so it was easier to read and fix later. This approach will help me in the future because it keeps big projects from feeling overwhelming.

**Testing Process**
I tested my app by running it in the emulator and trying different inputs like wrong passwords or empty fields. Testing is important because it shows where things break or don’t work like expected. It helped me find small mistakes early and fix them before finishing the project.

**Challenges and Innovation**
The hardest part was getting the login and database to work together correctly. I had to look back at examples and try different ideas until it worked. This pushed me to understand the code better instead of just copying it.

**Most Successful Part**
I think my strongest part was putting together a full working app from start to finish. From UI design to coding and testing, I showed I can follow the full development process. This project made me more confident in mobile app development and showed me how much I’ve learned in the program.
